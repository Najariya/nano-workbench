# Next-Version Store Asset Script

Use this script when refreshing screenshots or the YouTube promo for the next public update.

## Screenshot Order

| Order | Asset | Must show |
| --- | --- | --- |
| 1 | Workbench actions | Real side panel on a readable page, source label, read freshness, summarize action. |
| 2 | Right-click OCR | Right-click image menu with **Extract text/details with Local AI** and the OCR result copy controls. |
| 3 | Per-tab workspace | Two tabs with separate conversations and a visible Current tab / Pinned workspace label. |
| 4 | Privacy/local storage | Settings or privacy panel showing local storage and no developer data access. |
| 5 | Model setup | One-time local model setup/download/ready state in Chrome. |

## Promo Video Beats

| Beat | Duration | Visual |
| --- | --- | --- |
| Title | 2-3 seconds | Local AI Workbench name, Chrome side panel, local-first promise. |
| Page reading | 4 seconds | Open page, read freshness changes, summarize response with source label. |
| Image OCR | 5 seconds | Right-click image, extract text/fields, copy text/fields/JSON. |
| Workspace continuity | 4 seconds | Switch tabs and show separate workspace conversations. |
| Privacy | 4 seconds | Local storage/privacy note and no accounts/API keys. |

## Voiceover / Caption Line

Local AI Workbench brings on-device browser AI into Chrome: read pages, extract text from images, keep tab workspaces separate, and keep your work local.

## Quality Bar

- Use real screenshots from the extension, not generic mockups.
- Make the OCR context menu readable.
- Make the source label and read freshness visible.
- Avoid implying the developer can access user content.
- Do not show any removed experimental WebGPU option.
